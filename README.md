## 🧩 Challenges Encountered & How I Solved Them

| Challenge | What Went Wrong | How I Resolved It |
|-----------|----------------|-------------------|
| Cursor's Clone Repository button was unresponsive | Clicking the GUI button produced no result — no dialog, no prompt, nothing | Bypassed the GUI entirely and used Cursor's built-in terminal to run `git clone` manually |
| `cd Desktop` failed in the terminal | Windows with OneDrive remaps the Desktop to a non-standard path; the default `C:\Users\parin\Desktop` doesn't exist | Used `dir` to inspect the actual folder structure and navigated using the full OneDrive path instead |
| `git` command not recognized | Git is not bundled with Windows or Cursor — running `git clone` threw a `CommandNotFoundException` | Downloaded and installed Git from git-scm.com, then restarted the terminal so the system could register the new executable |
